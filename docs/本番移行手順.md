# EREYS DASH 本番移行手順書

デモ表示の状態から、**実データが保存される本番運用**へ切り替えるための手順です。
上から順に進めれば完了します。エンジニアでなくても進められるよう、画面の場所まで書いています。

> 所要時間の目安：Supabase＋Vercelで約30分／LINE連携も行う場合は＋30〜60分。

---

## 全体像（4ステップ）

```
① コードを本番に反映      → GitHubのPRをマージ
② データベースを用意       → Supabase作成 + schema.sql実行
③ 環境変数をVercelに設定   → DB接続・鍵・(LINE)
④ 反映と初期設定          → 再デプロイ → 管理者PW変更・店舗座標・スタッフ登録
```

現在は **Supabase 未設定＝デモモード**（入力しても保存されず、再起動で消える）です。
②③が終わって初めて、入力データが永続保存されます。

---

## ① コードを本番に反映する（GitHub）

本番（Vercel）は GitHub の `main` ブランチを自動デプロイします。未マージのPRがあれば先にマージします。

- [ ] https://github.com/vie324/eryes_dash/pulls を開く
- [ ] オープン中のPRの「Files changed」を確認し、緑の **Merge pull request** を押す
- [ ] マージ後、Vercel が自動でデプロイを開始する（数分）

> 補足：これまで「PRをマージした後に追加修正をプッシュ」したため、未反映のPRが残ることがありました。**pulls ページにオープンなPRが無い状態**にしてください。

## ② データベース（Supabase）を用意する

### 2-1. プロジェクト作成
- [ ] https://supabase.com にサインアップ／ログイン
- [ ] 「New project」を作成（Region は **Northeast Asia (Tokyo)** を推奨）
- [ ] 「Database Password」は自動生成でOK（このあとは使いません）。作成完了まで1〜2分待つ

### 2-2. テーブルを作成（schema.sql の実行）
- [ ] 左メニュー **SQL Editor** → 「New query」
- [ ] リポジトリの `supabase/schema.sql` の中身を**全文コピペ**
- [ ] 実行前に、ファイル末尾の初期データ（3店舗の名称・住所・緯度経度）を**実際の値に書き換える**
- [ ] **Run** を押す（テーブル・インデックス・初期店舗・初期管理者・シフトルールが作成される）

> これで管理者アカウント（ID: `admin` ／ PW: `admin1234`）も作られます。**④で必ず変更**します。

### 2-3. 接続情報を控える
- [ ] 左メニュー **Project Settings → API** を開く
- [ ] **Project URL**（`https://xxxx.supabase.co`）をコピー → 後で `SUPABASE_URL` に使う
- [ ] **Project API keys → `service_role`（secret）** をコピー → 後で `SUPABASE_SERVICE_ROLE_KEY` に使う

> ⚠ `service_role` キーは管理者権限の秘密鍵です。**サーバー専用**で、SNS・スクショ・クライアント側コードに出さないでください（本システムはサーバー内でのみ使用します）。

## ③ 環境変数をVercelに設定する

### 3-1. 秘密鍵を2つ生成する
PC のターミナル（Mac は「ターミナル」アプリ）で以下を実行し、出力をメモします。

```bash
openssl rand -hex 32   # → AUTH_SECRET 用（セッションCookieの署名鍵）
openssl rand -hex 16   # → CRON_SECRET 用（定時バッチの不正実行防止）
```

> ターミナルが使えない場合は、ランダムな英数字の長い文字列（32文字以上）でも構いません。

### 3-2. Vercel に登録
- [ ] https://vercel.com → 対象プロジェクト **eryes-dash** → **Settings → Environment Variables**
- [ ] 下表を **Production**（必要なら Preview も）に登録

| 変数名 | 値 | 必須 |
|---|---|---|
| `AUTH_SECRET` | 3-1で生成した32バイトの文字列 | ✅ |
| `CRON_SECRET` | 3-1で生成した16バイトの文字列 | ✅ |
| `SUPABASE_URL` | 2-3の Project URL | ✅ |
| `SUPABASE_SERVICE_ROLE_KEY` | 2-3の service_role キー | ✅ |
| `NEXT_PUBLIC_APP_URL` | 本番URL（例 `https://eryes-dash.vercel.app`） | 推奨 |
| `LINE_CHANNEL_ID` | LINEのチャネルID | LINE使用時 |
| `LINE_CHANNEL_SECRET` | LINEのチャネルシークレット | LINE使用時 |
| `LINE_CHANNEL_ACCESS_TOKEN` | LINEの長期アクセストークン | LINE使用時 |
| `NEXT_PUBLIC_LIFF_ID` | LIFF ID | LINE使用時 |

> LINE連携（カウンセリング・前日リマインド・一斉配信）を**今は使わない**場合、LINE系4つは空のままでOK。その間はLINE送信が「モック（画面ログのみ）」になります。後から追加できます。

### 3-3. 再デプロイ
- [ ] **Deployments** → 最新のデプロイ → 「Redeploy」（環境変数を反映させるため）

## ④ 反映後の初期設定（運用前に必ず）

本番URLにアクセスし、`admin` / `admin1234` でログインして設定します。

- [ ] **管理者パスワードを変更**（マスタ設定 → 自分のスタッフ → 新しいパスワード）
- [ ] **3店舗の住所・緯度経度・GPS許容半径**を確認（マスタ設定 → 店舗一覧）
      ※ 緯度経度は Google マップで店舗を右クリック → 出てくる数値をコピペ
- [ ] **スタッフを全員登録**（氏名・ログインID・初期パスワード・権限・固定残業時間）
- [ ] **シフトのルール設定**（連勤上限・各店舗の最低人数・希望締切日）を実運用に合わせる
- [ ] 画面右上「使い方」からヘルプを開き、スタッフに共有

---

## ⑤ LINE連携の設定（カウンセリング／リマインド／配信を使う場合）

詳細は `README.md`「4. LINE Developers 設定手順」を参照。要点のみ：

- [ ] LINE Developers で **Messaging APIチャネル**作成 → ID／シークレット／アクセストークンを③に設定
- [ ] Webhook URL に `https://<本番URL>/api/line/webhook` を設定し「Webhookの利用」をON →「検証」
- [ ] LINE公式アカウントの **応答メッセージをOFF**（本システムが自動応答するため）
- [ ] **LINEログインチャネル**で LIFF を作成（サイズ Full／エンドポイント `https://<本番URL>/liff/counseling`）→ LIFF IDを③に設定
- [ ] 公式アカウントの**リッチメニュー**に「カウンセリング」ボタンを作り `https://liff.line.me/<LIFF_ID>` を設定
- [ ] 設定後、③の環境変数を入れて再デプロイ

## ⑥ 定時バッチ（自動通知）の確認

`vercel.json` に設定済みのため、デプロイ時に自動で登録されます。

- [ ] Vercel → Settings → **Cron Jobs** に2件あることを確認
  - `/api/cron/reminder` … 毎日 **19:00 JST**（前日リマインド）
  - `/api/cron/shift-notice` … 毎月 **15日 10:00 JST**（シフト希望募集の通知）

> `CRON_SECRET` を設定済みなら、Vercel が自動で認証ヘッダーを付けて呼び出します（外部からの不正実行を防止）。

## ⑦ 動作確認チェックリスト（本番URLで）

- [ ] ログインできる（管理者・スタッフ）／変更後パスワードで入れる
- [ ] スタッフ：日報を保存 → ログアウト→再ログインしても**残っている**（＝DB保存OK）
- [ ] スタッフ：レジ締めを保存 → 管理者「成績・日報」に反映される
- [ ] GPS打刻：店舗付近で「打刻しました」、離れた場所で「圏外」表示
- [ ] 管理者：CSV出力（売上・現金）がExcelで文字化けせず開ける
- [ ] （LINE使用時）友だち追加 → 氏名送信 → 顧客一覧に登録される → カウンセリング送信 → スタッフ画面に表示
- [ ] スマホ・iPad の両方で表示崩れがない

## ⑧ 運用開始後のメンテナンス

- **データのバックアップ**：Supabase → Database → Backups（有料プランで自動。無料枠は手動エクスポート推奨）
- **LINE無料枠**：月の送信数はダッシュボードに表示。500通に近づいたら有料プラン検討（システム変更不要）
- **ロゴ差し替え**：`public/logo.png` を置き換える（白背景画像なら `node scripts/make-logo-transparent.mjs <画像>` で透過）
- **スタッフの追加・退職**：マスタ設定で追加／「有効」をオフ（退職者はオフにするとログイン不可・記録は保持）
- **CSV出力項目の変更**：税理士の指定が決まったら `src/app/api/admin/csv/route.ts` を修正

## よくあるトラブル

| 症状 | 原因と対処 |
|---|---|
| 入力しても保存されない・データが消える | Supabase未接続（デモモード）。③の環境変数を設定して再デプロイ |
| ログイン後すぐログアウトされる／ログインできない | `AUTH_SECRET` 未設定または変更で既存セッション無効。再ログインで解消 |
| 画面に「デモモードで動作中」の帯が出る | Supabase未接続のサイン。②③を確認 |
| 定時通知が来ない | Cron Jobsの登録、`CRON_SECRET` の一致、（リマインドは）対象予約の有無を確認 |
| LINEが送られない | LINE系環境変数の設定、Webhook URL／応答メッセージOFF、無料枠の残りを確認 |
| GPS打刻が常に圏外 | マスタ設定の店舗の緯度経度が正しいか、許容半径（既定100m）、端末の位置情報許可を確認 |
